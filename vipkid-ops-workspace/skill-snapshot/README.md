# vipkid-ops

`vipkid-ops` 是一个面向 VIPKID 国际化业务场景的运营 skill。它把商品包后台、家长详情页、Leads 线索库三套能力统一成一套可组合的工作方式，目标不是“记住很多接口”，而是“把业务问题快速拆成可执行步骤并给出结果”。

## 1. 这个 skill 解决什么问题

从高到低看，它主要解决 4 类问题：

1. 业务查询
   - 这个家长现在什么情况
   - 还有多少课
   - 有没有跟进记录
   - 这个节点从某天到现在有多少个家长
2. 后台配置
   - 新建 / 修改商品包
   - 调库存
   - 配优惠券限制
   - 配赠送权限
3. 线索处理
   - 查 Leads 私池 / 公海 / 冻结库
   - 批量分配 GCC / GCS / TMK
   - 流转到冻结库 / 公海
   - 标签增删、上传 Leads / 权益
4. 证据导出
   - 输出通话录音链接
   - 批量下载录音
   - 导出清单、失败列表、汇总文件

## 2. 这个 skill 的 top-down 结构

从最上层往下拆，这个 skill 由 7 个元能力组成：

### A. 身份与环境

负责判断：

- 该走 `sa-manager.lionabc.com` 还是 `sa-leads.lionabc.com`
- `token` 是否可用
- `cr_code` 是否正确

这是所有任务的起点。

### B. 对象定位

负责把业务说法翻译成目标对象和条件，比如：

- “未完课节点” => `WAIT_TRIAL_COMPLETE`
- “待付费” => `WAIT_PAY`
- “2.1 到现在” => 当年 `2月1日` 到今天
- “这个家长” => `parentId`
- “这批 leads” => `userIdList`

### C. 只读查询

负责拿数据，不改数据，包括：

- 商品包查询
- 家长详情 / 课时 / 学习行为 / 跟进 / 通话
- Leads 列表 / 节点统计 / 员工 / 标签 / 节点 / 渠道

### D. 聚合解释

负责把多份数据变成业务结论，比如：

- “这个家长没课时”
- “有 6 条录音”
- “注册时间从 2026-02-01 到现在，未完课节点有 148 个”

### E. 写操作

负责执行修改，包括：

- 商品包新建 / 修改 / 库存 / 优惠券 / 赠送权限
- 家长资料修改
- Leads 标签、分配、流转、上传

这层必须先确认影响范围，再执行。

### F. 批量处理

负责把单个动作扩展成批量任务，比如：

- 批量下载录音
- 批量分配 leads
- 批量导入 Excel
- 批量创建商品包

### G. 证据留存

负责保存结果，让业务可复查：

- 录音文件
- manifest 清单
- summary 汇总
- failures 失败清单

## 3. 组合方式

这 7 个元能力的意义，是方便组合。

常见组合是：

### 组合 1：业务问答

`身份与环境` + `对象定位` + `只读查询` + `聚合解释`

适合：

- “帮我看这个家长课时”
- “查一下跟进记录”
- “未完课节点从 2.1 到现在有多少个”

### 组合 2：先确认再修改

`身份与环境` + `对象定位` + `只读查询` + `写操作`

适合：

- “先帮我看这个商品包，再改价格”
- “确认这批家长后，再分给 GCC”

### 组合 3：批量取证

`身份与环境` + `对象定位` + `只读查询` + `批量处理` + `证据留存`

适合：

- “把这些家长的通话录音全部下载下来”
- “导出这批 leads 的清单”

## 7. 安全边界

这个 skill 明确不做：

- 商品包上架
- 商品包下架
- 删除类操作

这个 skill 做写操作前必须：

- 先确认对象存在
- 先展示影响范围
- 让用户二次确认

## 8. 适合谁用

这个 skill 最适合两类人：

- 业务同学：想直接拿结果、数字、链接、清单
- 运营 / 支持同学：需要把“查询 -> 判断 -> 执行 -> 留痕”串成一条链

如果用户只想要结果，就停留在“业务问答”层；如果用户要落地执行，再逐层进入写操作或批量处理。
